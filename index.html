How It Works: PlanSearch for Jokes

This app uses a method inspired by the PlanSearch paper. Instead of asking an LLM for a joke directly, we break down the creative process to generate more diverse and interesting results.

The Joke Generation Pipeline:

  1. 1. Generate Angles: Given your topic (e.g., "penguins"), we first ask the Generator LLM to come up with several diverse observations or "angles". (e.g., "they wear tuxedos", "they waddle", "they can't fly"). This creates a variety of starting points.
  2. 2. Create Setups (Plans): For each angle, we ask the LLM to create a joke setup. This is like the "plan" in PlanSearch. An angle like "they wear tuxedos" might lead to a setup like "Why did the penguin get a job as a waiter?".
  3. 3. Write Punchlines (Solutions): With a setup in hand, we ask the LLM to generate multiple punchlines (the "solutions"). This completes the joke.
  4. 4. LLM-as-a-Judge: We present all the generated jokes (in a random order to avoid bias) to a separate "Judge" LLM. The Judge rates each joke on a scale of 1-10 for funniness and explains its reasoning.
  5. 5. Showtime! The app displays the top-rated jokes from the judge.

On Novelty & Memorization

What is novelty in a joke? Novelty isn't just about being new; it's about creating a surprising, clever, and non-obvious connection between ideas. A truly novel joke isn't just a rehash of an old one with different words.

How would you test if a joke is truly novel? This is a hard problem! A multi-pronged approach would be best:

  • Plagiarism Check: Use search engines to look for the exact joke or key phrases online. If it's already on a hundred joke websites, it's not novel.
  • Semantic Search: Check the generated joke against a large database of existing jokes using semantic search, which looks for similar meanings, not just exact words.
  • Human Panel: The ultimate test. Ask a diverse group of people if they've heard the joke before and how original they find it.

How does this pipeline combat memorization? By breaking the process into `Angle -> Setup -> Punchline`, we force the LLM to construct a joke from conceptual building blocks rather than just recalling a complete joke it saw during training. Generating diverse angles first is key—it pushes the model into less-traveled creative territory, making simple memorization less likely to produce a coherent and funny result. It encourages combinatorial creativity over simple retrieval.
